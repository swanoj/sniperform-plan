3. How to Edit Pages

Option A — Edit directly on GitHub (easiest)

  1. Go to github.com/swanoj/sniperform-plan
  2. Click on the file you want to edit (e.g. sniperform_plan.html)
  3. Click the pencil icon (top-right of the file view) to edit
  4. Make your changes in the editor
  5. Scroll down, add a commit message (e.g. "Update pricing section")
  6. Click "Commit changes"
  7. Changes go live automatically within 1–2 minutes

Option B — Clone locally & push

# Clone the repo
git clone https://github.com/swanoj/sniperform-plan.git
cd sniperform-plan

# Edit files with any text editor / VS Code
code sniperform_plan.html

# Push changes
git add .
git commit -m "Update plan content"
git push origin master

Option C — Use GitHub CLI

# View repo
gh repo view swanoj/sniperform-plan

# Edit via API (for programmatic updates)
gh api repos/swanoj/sniperform-plan/contents/sniperform_plan.html

4. How GitHub Pages Works

Auto-deploy: Every time you push (or edit on GitHub), the site rebuilds automatically. No build step needed — these are plain HTML files served directly.
  • Source: master branch, root /
  • Any .html file in the repo root is accessible at https://swanoj.github.io/sniperform-plan/filename.html
  • To add a new page: just add a new .html file and push — it's live immediately
  • HTTPS is enforced automatically
  • No server, no CMS, no database — just static HTML files

5. Adding New Pages

  1. Create a new .html file (copy the structure from an existing page for consistent styling)
  2. Add it to the repo (push or upload via GitHub)
  3. Add a link to it from the sidebar nav of the main plan page
  4. It'll be live at https://swanoj.github.io/sniperform-plan/your-new-page.html

6. Page Structure Quick Reference

Both pages use the same styling pattern:

<div class="container">
    <div class="sidebar">
        <nav>
            <a href="#section-id">Section Name</a>
        </nav>
    </div>
    <div class="main">
        <h2 id="section-id">Section Title</h2>
        <p>Content here...</p>
    </div>
</div>

7. Access & Permissions

To give someone edit access: Go to the repo → Settings → Collaborators → Add people. They'll need a GitHub account.

Currently the repo owner is swanoj. Anyone added as a collaborator can push changes directly.
